# LibreTV 使用和打包说明

这份文档说明当前项目如何在本地运行、如何打包成 Android APK，以及安装到手机、平板、电视盒子时需要注意的地方。

## 1. 项目说明

当前项目包含两部分：

- Web 版本：使用 `index.html`、`player.html`、`js/`、`css/` 等前端文件运行。
- Android APK 版本：位于 `android/` 目录，通过 Android WebView 把 Web 页面打包进 APK。
- 直播电视：使用 `live.html` 进入，频道列表来自 `data/live-sources.json` 中配置的公开地址，也可以在页面里手动添加新的地址。

APK 版本不需要域名，不需要部署到 Vercel、Netlify、Render，也不需要运行 Node 服务。页面文件会被打包到 APK 里面，视频源请求和 m3u8 代理由 Android 原生层处理。

## 2. 本地 Web 运行

如果只是想在电脑浏览器里测试 Web 版本，可以在 VS Code 终端里执行：

```powershell
npm install
npm run start
```

然后浏览器打开：

```text
http://localhost:8080
```

电脑端直播页面：

```text
http://localhost:8080/live.html
```

也可以从首页点击“直播电视”进入。直播频道来自公开频道列表，页面里也可以添加自己的频道列表地址。

如果端口被占用，可以看终端输出的实际地址。

## 3. APK 打包环境

当前项目已经准备了本地打包工具，路径为：

```text
D:\xcx\tv\LibreTV-main\.android-tools
```

里面包含：

- JDK 17
- Android SDK
- Android Build Tools
- Gradle 8.7

这些工具只放在当前项目目录里，不会污染系统环境，也已经被 `.gitignore` 忽略，不会提交到代码仓库。

## 4. 在 VS Code 中打包 APK

1. 用 VS Code 打开项目目录：

```text
D:\xcx\tv\LibreTV-main
```

2. 打开 VS Code 终端。

3. 执行打包脚本：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\build-android-apk.ps1
```

4. 看到下面内容表示打包成功：

```text
BUILD SUCCESSFUL
APK generated: D:\xcx\tv\LibreTV-main\android\app\build\outputs\apk\debug\app-debug.apk
```

生成的 APK 文件位置：

```text
D:\xcx\tv\LibreTV-main\android\app\build\outputs\apk\debug\app-debug.apk
```

## 5. 安装 APK

### 安卓手机或平板

把下面这个 APK 复制到手机：

```text
D:\xcx\tv\LibreTV-main\android\app\build\outputs\apk\debug\app-debug.apk
```

然后在手机上点击安装。如果系统提示不允许安装未知来源应用，需要在系统设置里允许当前文件管理器或浏览器安装 APK。

### 安卓电视盒子

可以通过 U 盘、局域网传输工具、ADB 等方式安装。

如果使用 ADB，先确认设备已经连接：

```powershell
.\.android-tools\android-sdk\platform-tools\adb.exe devices
```

安装 APK：

```powershell
.\.android-tools\android-sdk\platform-tools\adb.exe install -r .\android\app\build\outputs\apk\debug\app-debug.apk
```

电视盒子上可能需要打开：

- 开发者选项
- USB 调试
- 允许安装未知来源应用

## 6. 鸿蒙系统说明

如果是支持 Android APK 兼容模式的鸿蒙设备，可以尝试直接安装这个 APK。

如果是 HarmonyOS NEXT，则不能直接安装 Android APK，需要单独开发鸿蒙原生应用包。

## 7. 修改代码后怎么重新打包

修改 Web 页面、播放器逻辑、样式或视频源逻辑后，直接重新执行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\build-android-apk.ps1
```

打包时会自动把根目录里的 Web 文件同步到 APK 资源目录，不需要手动复制。

不要直接修改下面这个目录里的生成文件：

```text
android\app\src\main\assets\site
```

应该修改根目录里的源码文件，例如：

- `index.html`
- `player.html`
- `css/`
- `js/`
- `libs/`

## 8. 当前 APK 的特点

- 不需要域名
- 不需要服务器部署
- 不需要 Node 在手机或电视盒子里运行
- 支持 WebView 播放页面
- 支持视频全屏
- 支持电视盒子入口
- 支持直播电视入口
- 支持添加自己的直播频道列表地址
- 支持 Android 返回键退出全屏或返回上一页
- 内置原生代理处理接口、m3u8、key、map、视频分片等请求

## 9. 清晰度相关功能

当前项目已经加入：

- 画质增强开关
- 多种画质增强模式
- WebGL 锐化模式
- 高清源识别
- 换源时优先标记更可能是高清的源
- hls.js 多清晰度源更积极切到高清

需要注意：这些功能只能改善显示效果和选择策略，不能把低码率视频源真正变成原生 1080P。如果视频源本身只提供 480P 或低码率内容，前端只能做锐化、对比度、饱和度等观感优化。

## 10. 常见问题

### 打包脚本提示不能运行

可以使用下面命令运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\build-android-apk.ps1
```

### APK 安装失败

常见原因：

- 设备不允许安装未知来源应用
- 设备 Android 版本太旧
- 旧版本 APK 签名冲突

可以先卸载旧版再安装新版。

### 视频不能播放

可能原因：

- 视频源失效
- 视频源限制地区、Referer 或 User-Agent
- 盒子系统 WebView 太旧
- 网络无法访问对应视频源

可以先在电脑浏览器 Web 版本里测试同一个视频源，再判断是不是 APK 问题。

### 直播频道打不开

可能原因：

- 这个频道地址已经失效
- 当前网络访问不了这个频道
- IPv6 频道需要网络支持 IPv6
- 电视盒子的系统 WebView 太旧

可以先点“刷新”，再换一个频道列表或换一个频道试试。

### 清晰度没有 1080P / 720P / 480P

这通常说明视频源没有提供多清晰度 m3u8 master playlist，或者源站没有暴露清晰度信息。项目可以识别和优先选择更可能高清的源，但不能凭空生成源站没有提供的清晰度档位。

## 11. 正式发布说明

如果要正式分发，建议后续再做：

- 修改应用包名

- 生成 release APK
- 使用自己的签名证书签名
- 测试手机、平板、电视盒子兼容性
